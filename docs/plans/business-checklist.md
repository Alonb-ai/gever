## היעד: 100 משתמשים משלמים בחצי השנה הראשונה

מביא מעל נקודת-האיזון לרווח (₪400-800/חודש). אבני-דרך:

| חודש | משלמים | פוקוס |
|---|---|---|
| 0-1 | 0 | **Phase 0** — עוסק, אימות מטא→מספר אמיתי, Lemon Squeezy, ToS. **הכרטיס לכניסה.** |
| 2 | ~10 | לאמת willingness-to-pay (משלמים באמת, לא רק אוהבים בחינם) + wow + onboarding |
| 3-4 | ~40 | **איזון.** referral + תוכן-מייסד |
| 5-6 | ~100 | הגברת הלולאה |

**סיכונים (לא הכלכלה — היא מצוינת):** (1) מספר-אמיתי חסום על עוסק=#1; (2)
willingness-to-pay; (3) retention. **הצעד שמתחיל את השעון: לפתוח עוסק עכשיו.**

## אתגר-ליבה: תדירות × כאב (למה לא להוביל עם מסעדות)

מנוי ₪15/חודש דורש **ערך תכוף או כואב**. מסעדה = נדירה (~1-3/חודש) **וגם** קלה
לעשות לבד → הרביע הכי חלש למונטיזציה. דמו מעולה, עוגן חלש.

|  | כאב נמוך | כאב גבוה |
|---|---|---|
| **תדיר** | משלוחים/סופר (קל באפליקציה) | חניה, תורים (רופא/מספרה) |
| **נדיר** | 🔴 מסעדות, קולנוע (הדגל היום) | ✅ ביטוח, טפסים ממשלתיים, ביטול-מנויים, עררים, מחלוקות-חיוב |

**הפיבוט:** גבר = "מי שעושה את העבודה הדיגיטלית המעצבנת" (מילוי-טפסים = הסופר-פאוור).
ורטיקלים כאב-גבוה מצדיקים ₪15 גם בנדירות (מודל "מנוי-שקט-נפשי" כמו ביטוח).
- העוגן צריך להיות ורטיקל כאב-גבוה, לא מסעדות (מסעדות = דמו-כניסה ויראלי בלבד).
- הימור: ביטול-מנויים + תורים לרופא/קופ"ח כ-1-2 הורטיקלים הראשונים לבחון.
- גלובלי+אנגלית = משחק-סקייל, לא תדירות. **לא עכשיו** — קודם לפצח את המודל בשוק אחד.
- ביטוח 15-20 דק' = בעיית UX/מהירות (retention), נפרדת מתדירות.
- לשקול מחדש: אם רוב השימוש נדיר, אולי pay-per-task/freemium מתאים מ-₪15 קבוע.

## שרשרת-התלויות הקריטית (למה עוסק ראשון)

```
עוסק פטור  ──►  אימות עסק מטא  ──►  מספר WhatsApp אמיתי
     │
     ├──►  Lemon Squeezy (דורש ישות עסקית)
     └──►  חשבוניות ללקוחות
```

העוסק הוא **צוואר-הבקבוק היחיד** שחוסם מספר-אמיתי, תשלומים וחשבוניות בבת-אחת.

## Unit Economics — עלות פר-הזמנה (נמדד 23.7.26, ממחירים מאומתים)

| רכיב | כמות/ריצה | מחיר | עלות/ריצה |
|---|---|---|---|
| Browserbase | ~3.3 דק' (0.055 שעה) | $0.10-0.12/שעה | ~$0.006 |
| Gemini נווט (flash) | ~25 צעדים × (~7K input + צילום) | $0.25/M in, $1.50/M out | ~$0.05 |
| Gemini שיחה | כמה קריאות קטנות | זניח | ~$0.005 |
| WhatsApp | הכל בתוך חלון 24ש' | חינם | $0 |
| **סה"כ** | | מסעדה/קולנוע ~$0.06 · ביטוח ~$0.12 | **~₪0.20-0.45** |

**מסקנות:**
1. עלות שולית זעירה → מרווח ענק אפילו במחיר צנוע (₪5/הזמנה או ₪30/חודש = ×20-100).
2. Gemini הנווט = ~80% מהעלות (25 קריאות, DOM גדול + צילומים). יעד הזירוז העתידי.
3. WhatsApp חינם בזרימת-ההזמנה — רק follow-up מחוץ ל-24ש' עולה.

**סייגים:**
- **ריצת-כישלון עולה אותו דבר** בלי לספק ערך. אם SLA="משלם רק על הצלחה" →
  אחוז-ההצלחה = המרווח. זו החוליה SLA↔כלכלה.
- אי-ודאות: נפח טוקני Gemini (אומדן מגדלי-state 18-40KB/צעד); מדויק בדשבורד AI Studio.

## נקודת האיזון — מודל ₪15/חודש (עלויות אמיתיות, מחמיר)

**קבוע/חודש:** Browserbase $20 + Elestio $15 + Supabase $25 = **$60 ≈ ₪230**
(Gemini/WhatsApp ללא-קבוע, DuckDNS חינם).

**תרומה/משתמש:** ₪15 − ₪2.7 עמלת Lemon Squeezy = ₪12.3 נטו − ₪6 שולי (מחמיר:
~10 הזמנות/חודש + 30% כישלון) = **₪6.3**.

**נקודת איזון:** ₪230 ÷ ₪6.3 ≈ **37 משתמשים** (מחמיר) · ~22 (שימוש קליל).
בטווח 40-60 משתמשים — כבר ברווח. Browserbase $20 מחזיק עד ~180 משתמשים בלי שדרוג.

| משתמשים | רווח מחמיר (₪6.3) | רווח קליל (₪10.3) |
|---|---|---|
| 50 | ₪85 | ₪285 |
| 100 | ₪400 | ₪800 |
| 500 | ₪2,920 | ₪4,920 |
| 1,000 | ₪6,070 | ₪10,070 |

**₪15 = משחק-נפח.** רווח משמעותי בסקייל (500-1000); מתחת ל-100 מכסה-עלויות-ומעט.

## פרסום ממומן = CAC, לא עלות קבועה (הסיכון של מודל ₪15)

מחירי ישראל (Tier 1, יקר): Meta/IG CPM $5-18/CPC ~$1-3.3 · TikTok CPM $5-12.
**עלות לקוח משלם (מחמיר): ~₪80-120.**

**תקופת החזר: CAC ₪80 ÷ תרומה ₪6.3 = ~13 חודשים** (קליל: ~8). אם לקוח עוזב
לפני כן — הפסד. פרסום ₪1,000/חודש → ~12 לקוחות → +₪76/חודש בלבד: הפרסום **דוחה**
רווחיות, לא מזרז, במחיר הזה.

**מסקנה:** ₪15 = משחק **אורגני/referral** (CAC≈0, WhatsApp ויראלי), לא משחק-פרסום.
פרסום ממומן רק כתקציב-ניסוי קטן ללמוד מסר. כדי שפרסום יעבוד כמנוע → CAC<₪40 או מחיר גבוה יותר.

## הפריטים הקיומיים שקל לשכוח (הדגשה)

1. **יחידת-כלכלה** — עלות פר-הזמנה מול מחיר. בלי זה, סקייל = הפסד גדול יותר.
2. **משפטי** — ToS + פרטיות + הגבלת-אחריות. גבר פועל עם PII ומבצע פעולות בשם הלקוח.
3. **זיהוי-תקלות** — ניטור + התראה + runbook. גיבוי מציל *אחרי* אסון; ניטור מונע.
4. **תמחור** — ההחלטה קודמת להקמת Lemon Squeezy.
