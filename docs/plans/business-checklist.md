## Phase 0 — לפני הלקוח המשלם הראשון (חסמים ויסודות)

- ☐ **עוסק פטור** — צוואר הבקבוק. פותח את כל השאר.
- ☐ **אימות עסק מטא** → מספר WhatsApp אמיתי (חסום על העוסק; ה-WABA כבר מחכה)
- ☐ **משפטי — קיומי:**
  - ☐ תנאי שימוש (ToS) עם **הגבלת אחריות** (גבר פועל בשם הלקוח)
  - ☐ מדיניות פרטיות (חוק הגנת הפרטיות + אולי רישום מאגר מידע — להתייעץ עו"ד)
  - ☐ מיפוי האזור-האפור: תנאי-שימוש של אתרי היעד (Ontopo/הוט) לגבי בוטים
- ☐ **החלטת מודל-תמחור** (מנוי / פר-הזמנה / קרדיטים / freemium) — *קודמת* ל-Lemon Squeezy
- ☐ **Lemon Squeezy** — הקמה לפי המודל שנבחר
- ☑ **יחידת-כלכלה — נמדד 23.7.26** (ראה למטה). עלות שולית ~₪0.20-0.45/הזמנה.
- ☐ **ניטור-uptime + התראה** — זיהוי תקלה בפרוד לפני הלקוח (יש גיבוי, אין זיהוי)
- ☐ **גיבויים** — קוד (git/remote) + DB (Supabase) + סודות (.env / ENCRYPTION_KEY)

## Phase 1 — תפעול בטא

- ☐ קבוצת פידבק מלקוחות
- ☐ רוטינות ניטור: חריגות-פרסונה + כשלי-ריצות (תיעוד אוטומטי)
- ☐ מערך תמיכה — מי עונה? שעות? נתיב הסלמה?
- ☐ **Audit log** — מה גבר עשה, למי, עם איזה נתונים (דיבוג + מחלוקות)
- ☐ **הגדרת הבטחה/SLA** — "מנסים; נכשלנו — לא משלם"? מנהל ציפיות + מגן משפטית
- ☐ **Onboarding** למשתמש חדש — הרגע הראשון קובע retention
- ☐ **מדיניות ביטול/החזר** + תהליך (חוק הגנת הצרכן)
- ☐ **מניעת שימוש-לרעה** — rate-limit פר-משתמש, טיפול בבקשות מזיקות/לא-חוקיות
- ☐ **דשבורד KPI** — הפעלה, retention, אחוז-הצלחה, עלות/הזמנה, הכנסה
- ☐ בדיקת אבטחה + אודיט + הקשחת מנגנוני אבטחה

## Unit Economics — עלות פר-הזמנה (נמדד 23.7.26, ממחירים מאומתים)

| רכיב | כמות/ריצה | מחיר | עלות/ריצה |
|---|---|---|---|
| Browserbase | ~3.3 דק' (0.055 שעה) | $0.10-0.12/שעה | ~$0.006 |
| Gemini נווט (flash) | ~25 צעדים × (~7K input + צילום) | $0.25/M in, $1.50/M out | ~$0.05 |
| Gemini שיחה | כמה קריאות קטנות | זניח | ~$0.005 |
| WhatsApp | הכל בתוך חלון 24ש' | חינם | $0 |
| **סה"כ** | | מסעדה/קולנוע ~$0.06 · ביטוח ~$0.12 | **~₪0.20-0.45** |

**מסקנות:**
1. עלות שולית זעירה → מרווח ענק אפילו במחיר צנוע (₪5/הזמנה או ₪30/חודש = ×20-100).
2. Gemini הנווט = ~80% מהעלות (25 קריאות, DOM גדול + צילומים). יעד הזירוז העתידי.
3. WhatsApp חינם בזרימת-ההזמנה — רק follow-up מחוץ ל-24ש' עולה.

**סייגים:**
- **ריצת-כישלון עולה אותו דבר** בלי לספק ערך. אם SLA="משלם רק על הצלחה" →
  אחוז-ההצלחה = המרווח. זו החוליה SLA↔כלכלה.
- עלויות קבועות נפרדות (Browserbase $20 + Elestio + Supabase). בנפח נמוך הן השולטות.
- אי-ודאות: נפח טוקני Gemini (אומדן מגדלי-state 18-40KB/צעד); מדויק בדשבורד AI Studio.

## הפריטים הקיומיים שקל לשכוח (הדגשה)

1. **יחידת-כלכלה** — עלות פר-הזמנה מול מחיר. בלי זה, סקייל = הפסד גדול יותר.
2. **משפטי** — ToS + פרטיות + הגבלת-אחריות. גבר פועל עם PII ומבצע פעולות בשם הלקוח.
3. **זיהוי-תקלות** — ניטור + התראה + runbook. גיבוי מציל *אחרי* אסון; ניטור מונע.
4. **תמחור** — ההחלטה קודמת להקמת Lemon Squeezy.
