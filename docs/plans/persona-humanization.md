# פיצ'ר: האנשת הפרסונה — שגיאות כתיב + קצב הודעות

> תועד 24.7.2026 (בקשת אלון). סטטוס: **מתוכנן — לא בפיתוח.** (Phase E7 ב-roadmap)

## 1. שגיאות כתיב מכוונות + תיקון-עצמי

**הרעיון:** פה ושם גבר שולח מילה עם טעות הקלדה אנושית, ומיד אחריה הודעת תיקון
עם כוכבית — כמו כולם בוואטסאפ:

```
מןכן          ← נ' סופית באמצע מילה
מוכן*
```
```
תבדוק לי מקופ
מקום*
```

זה מכניס חיים לשיחה ומחזק את תחושת האדם-מאחורי-המקלדת.
(הדוגמאות להמחשה בלבד — **אין מחסן מילים**; המודל מייצר את הטעות בעצמו,
בכל מילה יומיומית שיוצא לו, כמו אדם אמיתי.)

**ההגדרה (אלון, 24.7):**
- **עדין מאוד: גג פעם-פעמיים בשיחה שלמה.** לא יותר.
- **הטעות באמצע הודעה גדולה** — מילה שגויה בתוך משפט זורם, לא הודעה קצרה
  שכולה הטעות. ככה זה נראה אמיתי.
- **התיקון תמיד בהודעה הבאה**, ונשמע טבעי (`מוכן*` — כמו כולם בוואטסאפ).
- **לעולם לא בנתונים:** שמות מקומות, שעות, תאריכים, מספרים, סכומים, לינקים,
  ורשימות אופציות — נקיים תמיד. טעות ב"21:30" או בשם מסעדה היא באג, לא קסם.

**כיוון מימוש — "מתי" בקוד, "איך" בפרסונה:**
- **ה"איך" — הוראה ברמת הפרסונה, בלי מחסן מילים:** תיאור-אופי (character-driven,
  כמו כל הפרסונה): "לפעמים, באמצע הודעה ארוכה, יוצאת לך טעות הקלדה קטנה במילה
  יומיומית — ואתה מתקן אותה בהודעה הבאה עם כוכבית, בלי לעשות מזה עניין."
  המודל בוחר את המילה ואת הטעות בעצמו.
- **ה"מתי" — שער דטרמיניסטי בקוד:** ההוראה לא יושבת קבוע בפרומפט (המודל יגזים).
  ה-pipeline מזריק אותה לתור בודד (מנגנון ההזרקה הנקודתי הקיים, כמו
  feedback_sentiment), עם מונה פר-שיחה ב-state: אחרי 1-2 הפעלות — ההוראה לא
  מוזרקת יותר באותה שיחה. גם ההגרלה מוטה לתורות של הודעות ארוכות/שיחתיות
  (לא תורות של אופציות/אישורי הזמנה).
- גארד קוד: אם ההודעה שנוצרה מכילה את הטעות בשורת אופציה/מספר/שם — מוותרים.

## 2. ריווח קצב ההודעות (pacing)

**הבעיה:** פיצול-ההודעות של הפרסונה נשלח היום ברצף מיידי — גבר "מפציץ" את
הצ'אט, ואלמנט ההקלדה-האנושית הולך לאיבוד (ראה צילום השיחה מ-23.7: 6 הודעות
באותה דקה).

**הפתרון:** השהיה קצרה בין הודעות עוקבות באותו burst — לא מוגזמת:
- בסיס ~800-1500ms בין הודעות, מדורג לפי אורך ההודעה הבאה (הודעה ארוכה =
  "לוקח זמן להקליד אותה"). ג'יטר קטן אקראי שלא ירגיש מטרונום.
- תקרה כוללת ל-burst (למשל 5-6 שניות סה"כ) — לא מעכבים באמת את הלקוח.
- קונפיג ב-settings (`MESSAGE_PACING_MS`), כבוי=0 לטסטים.
- הודעות סטטוס דחופות (אישור הזמנה, שגיאה) — בלי השהיה מלאכותית.
- לבדוק: WhatsApp Cloud API תומך ב-typing indicator? אם כן — להדליק בזמן
  ההשהיה, זה מוכר את האפקט כולו. (לא מאומת — לבדוק בתיעוד Meta לפני מימוש.)

**תלות:** משלים את תיקון הירי-הכפול (E1/#3) — קודם מפסיקים לשלוח כפול,
אחר-כך מרווחים את מה שכן נשלח.
